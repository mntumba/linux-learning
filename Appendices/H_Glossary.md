# Appendix H: Glossary

**ACL (Access Control List)** – A list of permissions attached to an object specifying which users or system processes can access it.

**ARP (Address Resolution Protocol)** – Maps IP addresses to MAC addresses on a local network.

**Authentication** – The process of verifying the identity of a user or system.

**Authorization** – The process of determining what permissions an authenticated user has.

**Bash** – Bourne Again SHell; the default shell on most Linux distributions.

**Binary** – An executable file in machine-readable format; also refers to base-2 number representation.

**Bootloader** – Software that loads the operating system (e.g., GRUB on Linux).

**Buffer Overflow** – A vulnerability where a program writes more data to a buffer than it can hold, potentially overwriting adjacent memory.

**CIDR (Classless Inter-Domain Routing)** – A method for allocating IP addresses using a prefix length (e.g., 192.168.1.0/24).

**CLI (Command Line Interface)** – A text-based interface for interacting with the operating system.

**Cron** – A time-based job scheduler in Unix/Linux that runs commands at specified intervals.

**CVE (Common Vulnerabilities and Exposures)** – A standardized identifier for publicly known cybersecurity vulnerabilities.

**CVSS (Common Vulnerability Scoring System)** – A numerical score (0-10) representing the severity of a vulnerability.

**Daemon** – A background process that runs without direct user interaction (e.g., sshd, httpd).

**DHCP (Dynamic Host Configuration Protocol)** – Automatically assigns IP addresses to devices on a network.

**DNS (Domain Name System)** – Translates human-readable domain names to IP addresses.

**ELF (Executable and Linkable Format)** – The standard binary format for executables on Linux.

**Environment Variable** – A dynamic named value that can affect the way running processes behave (e.g., PATH, HOME).

**Exploit** – Code or a technique that takes advantage of a vulnerability to cause unintended behavior.

**File Descriptor** – An integer handle used to reference open files or I/O streams (0=stdin, 1=stdout, 2=stderr).

**Filesystem** – The method and data structure used to organize and store files (e.g., ext4, XFS, Btrfs).

**Firewall** – A network security device or software that monitors and controls incoming and outgoing network traffic.

**Fork** – Creating a new process as a copy of an existing one using the fork() system call.

**GPG (GNU Privacy Guard)** – A free implementation of the OpenPGP standard for encryption and signing.

**GRUB (Grand Unified Bootloader)** – The default bootloader for most Linux distributions.

**Hash** – A fixed-size output produced by a hash function from variable-length input (e.g., MD5, SHA-256).

**Inode** – A data structure on a filesystem storing metadata about a file (permissions, timestamps, pointers to data blocks).

**IDS/IPS (Intrusion Detection/Prevention System)** – Systems that monitor network or host activity for malicious activity.

**Inode** – A file system structure holding file metadata (owner, permissions, timestamps, data block pointers).

**iptables** – A user-space utility for configuring the Linux kernel's packet filtering framework (Netfilter).

**Kernel** – The core component of the operating system that manages hardware and system resources.

**LDAP (Lightweight Directory Access Protocol)** – A protocol for accessing and maintaining distributed directory information.

**LVM (Logical Volume Manager)** – A device mapper providing logical volume management for the Linux kernel.

**MAC Address** – A unique hardware identifier assigned to a network interface card.

**Metasploit** – An open-source penetration testing framework for developing and executing exploits.

**Mounting** – Attaching a filesystem to a directory in the filesystem tree.

**NAT (Network Address Translation)** – Remaps IP addresses by modifying IP header information in packets.

**NFS (Network File System)** – A distributed file system protocol for sharing files over a network.

**OSINT (Open Source Intelligence)** – Intelligence gathered from publicly available sources.

**PAM (Pluggable Authentication Modules)** – A framework for authentication in Linux.

**Payload** – In security, the component of malware or an exploit that performs the malicious action.

**Permissions** – Rules determining who can read, write, or execute a file (rwx for owner/group/other).

**PID (Process ID)** – A unique integer assigned to each running process.

**Pipe** – A mechanism for passing the output of one command as input to another (|).

**Port** – A logical construct identifying a specific process or network service on a computer.

**Privilege Escalation** – Gaining higher-level privileges than originally assigned (horizontal or vertical).

**Process** – A running instance of a program.

**RAID (Redundant Array of Independent Disks)** – A data storage virtualization technology combining multiple disks.

**Regex (Regular Expression)** – A sequence of characters defining a search pattern.

**Root** – The superuser account with full administrative privileges on a Linux system.

**SUID (Set User ID)** – A permission bit that allows a file to run with the privileges of the file owner.

**SSH (Secure Shell)** – A cryptographic network protocol for secure remote login and command execution.

**SSL/TLS** – Protocols for establishing authenticated and encrypted links between computers.

**Subnet** – A logical subdivision of an IP network.

**Sudo** – A command allowing permitted users to run commands as another user (typically root).

**Swap** – Disk space used as virtual memory when RAM is full.

**Symlink (Symbolic Link)** – A file that points to another file or directory.

**Syslog** – A standard for message logging in Linux systems.

**Systemd** – A modern init system and service manager for Linux.

**TCP/IP** – The foundational communication protocols of the internet.

**TTY** – Teletypewriter; refers to a terminal in Linux.

**UID (User ID)** – A unique integer identifying a user account (root = 0).

**VLAN (Virtual LAN)** – A logical subdivision of a network at layer 2.

**VPN (Virtual Private Network)** – Creates a secure, encrypted tunnel over a public network.

**Vulnerability** – A weakness in a system that can be exploited to gain unauthorized access.

**Wildcard** – A character that represents one or more other characters in a command (* matches any string).

**X.509** – A standard defining the format of public key certificates used in TLS/SSL.

**YARA** – A tool for identifying and classifying malware samples based on textual or binary patterns.

**Zero-Day** – A vulnerability unknown to the software vendor with no available patch.
